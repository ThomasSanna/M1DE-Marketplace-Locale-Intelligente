## Sujet du projet

Dans le cadre de votre projet pour le semestre vous devez concevoir et développer une application complète avec :

— Une architecture Front-end / Back-end ;
— Une infrastructure conteneurisée (Docker) ;
— Un pipeline CI/CD ;
— Un déploiement sur une infrastructure Cloud ;
— Une solution de monitoring et observabilité ;
— Une démarche SRE (Site Reliability Engineering) ;
— Une exploitation des données par une équipe Data Analyst ;

Le projet est à réaliser en équipe jusqu'à la date des présentation prévue pour le 00/00/2026. L'objectif du projet sera de mettre en oeuvre tout ce qu'on a vu et qu'on vera tout au long du semestre. Le projet permettra donc de valider votre capacité à travailler au sein d'une équipe agile adoptant la méthodologie SCRUM, avec des sprints structurés, des réunions de planification, des revues et des rétrospectives. Le projet devra être géré via Git en appliquant un workflow structuré de type GitFlow, incluant la gestion de branches, les Pull Requests et les revues de code systématiques. Une intégration et un déploiement continus (CI/CD) devront être mis en place afin d'automatiser les phases de tests, de build et de déploiement. L'ensemble de l'application devra être conteneurisé à l'aide de Docker, avec orchestration via docker-compose ou Kubernetes. Le déploiement devra être réalisé sur une infrastructure Cloud (AWS, Azure ou GCP). Les étudiants devront également mettre en œuvre une solution de monitoring telle que Prometheus ou Grafana afin de superviser l'application et ses services. Une démarche SRE sera attendue, incluant la définition d'indicateurs de fiabilité tels que les SLO, SLA et l'Error Budget. Enfin, les données générées par l'application devront faire l'objet d'une exploitation analytique permettant de produire des indicateurs, des tableaux de bord et des analyses pertinentes.

— Travailler au sein d'une équipe agile utilisant SCRUM,
— Gérer un projet via Git (GitFlow, Pull Requests, Code Review),
— Mise en place d'un pipeline CI/CD automatisé,
— Conteneurisation et orchestration (docker-compose ou Kubernetes),
— Déploiement Cloud (AWS / Azure / GCP),
— Mise en œuvre d'outils de monitoring (Prometheus ou Grafana)
— Définition d'indicateurs SRE (SLO, SLA, Error Budget)
— Exploitation analytique des données générées

Les données générées par votre application devront être exploitables par une équipe Data. Vous devrez :

— Fournir un accès SQL ou API aux données
— Proposer au minimum 3 analyses pertinentes

Exemples d'analyses possibles :

— Analyse comportement utilisateur
— Détection d'anomalies
— Modèle prédictif simple
— Segmentation (clustering)

Un tableau de bord analytique par exemple pourrait être produit avec PowerBI, Metabase, Superset ou équivalent.

## Organisation d'équipe

Chaque équipe (5 à 6 étudiants) devra répartir les rôles :

— Lead Developer
— DevOps Engineer
— Data Engineer / Data Analyst
— Product Owners
— Scrum master

Un(e) même étudiant(e) peut avoir plusieurs rôles.

## Sujet de notre projet — Marketplace locale intelligente

Ce projet vise à développer une place de marché en ligne destinée aux producteurs locaux. L'application devra inclure un système de gestion des stocks, des commandes et des paiements simulés. Les données générées permettront d'analyser le comportement d'achat, de réaliser une segmentation client (clustering) et de produire des indicateurs de performance commerciale.
